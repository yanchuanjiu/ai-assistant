# SKILL: IT 项目集管理 (Portfolio & Project Management)

> 适用领域：电商、数字化转型、AI/大模型、数据平台，多部门协同、多系统集成类项目。
> 触发时机：任何涉及项目状态、需求、方案、会议纪要、进展汇报、风险、决策等写入飞书的场景。

---

## 一、飞书知识库页面结构规范

所有项目文档统一在飞书知识库中按以下层级组织：

```
📁 项目集 (Portfolio)
  ├── 📄 项目集概览          ← 所有项目一览、健康度、关键里程碑
  └── 📁 [项目代号] [项目名] [状态emoji]
        ├── 📄 00_项目章程
        ├── 📄 01_需求与范围
        ├── 📄 02_技术方案
        ├── 📄 03_项目计划
        ├── 📁 04_会议纪要    ← 每次会议一条记录，追加写入
        ├── 📄 05_状态周报    ← 每周追加，最新在最上方
        ├── 📄 06_RAID 日志  ← 风险/行动项/问题/决策，持续维护
        └── 📄 07_项目复盘    ← 项目结束后填写
```

**页面命名规则**：`[序号]_[文档类型]`，如 `02_技术方案`。
**文件夹命名规则**：`[项目代号] [项目中文名] [状态emoji]`，如 `AIKG 知识库AI项目 🟢`。

### 项目状态 Emoji
| Emoji | 状态 | 说明 |
|-------|------|------|
| 🔵 | 规划中 | 立项尚未启动 |
| 🟢 | 进行中 | 正常推进 |
| 🟡 | 有风险 | 存在阻塞或延期风险 |
| 🔴 | 严重阻塞 | 需要升级处理 |
| ⏸️ | 暂停 | 外部原因暂停 |
| ✅ | 已完成 | 上线或验收通过 |

## 三、写入飞书前的治理检查

**在执行任何项目相关的飞书写入操作之前，先检查：**

1. **页面归属正确**：内容属于哪个项目？写到对应项目文件夹下的对应文档（不要全部堆到会议纪要汇总页）
2. **文档类型匹配**：会议纪要→`04_会议纪要`，风险问题→`06_RAID 日志`，进展→`05_状态周报`，决策→`06_RAID 日志`中的 Decisions 块
3. **使用对应模板**：追加格式的文档（周报/会议纪要/RAID）用追加写入；章程/方案用覆盖写入
4. **结构完整性**：写入内容应包含模板中的关键字段，不要只写碎片信息

**例外**：临时笔记、草稿、碎片信息可先放到"📋 会议纪要汇总"临时页，但在整理时应归位到正确项目文档。

## 五、AI/电商/数字化项目特殊关注点

- **多系统接口**：每个系统集成点必须在 `02_技术方案` 的"系统交互清单"中登记
- **数据治理**：涉及用户数据、订单数据的项目，技术方案必须包含数据分级和合规说明
- **AI 项目**：模型版本、训练数据来源、评估指标必须记录；上线后监控方案必须明确
- **多部门协同**：RACI 矩阵必须填写，避免权责不清；跨部门决策必须记录在 RAID 日志的 Decisions 块
- **上线管理**：发布前核查清单（Checklist）应在 `03_项目计划` 或独立页面中维护
