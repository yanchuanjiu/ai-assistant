# SKILL: IT 项目集管理（摘要）

> 完整文档模板见飞书知识库或本地 `workspace/SKILLS_PROJECT_MGMT_TEMPLATES.md`（需要时用 `feishu_read_page` 或直接读取）。

---

## 一、飞书知识库页面结构

```
📁 [项目代号] [项目名] [状态emoji]
  ├── 📄 00_项目章程
  ├── 📄 01_需求与范围
  ├── 📄 02_技术方案
  ├── 📄 03_项目计划
  ├── 📁 04_会议纪要    ← 每次会议追加
  ├── 📄 05_状态周报    ← 每周追加，最新在最上方
  ├── 📄 06_RAID 日志  ← 风险/行动项/问题/决策
  └── 📄 07_项目复盘
```

**状态 Emoji**：🔵规划中 🟢进行中 🟡有风险 🔴严重阻塞 ⏸️暂停 ✅已完成

## 二、快速索引：用户意图 → 工具 → 关键参数

| 用户意图 | 调用工具 | 关键参数 |
|----------|----------|----------|
| 新建项目 | `feishu_project_setup` | `project_name`, `project_code` |
| 写会议纪要 | `feishu_wiki_page(find_or_create)` → `feishu_append_to_page` | parent = `04_会议纪要` token |
| 更新风险/决策/行动项 | `feishu_append_to_page` | target = `06_RAID 日志` token |
| 写周报/进展 | `feishu_append_to_page` | target = `05_状态周报` token，最新在上 |
| 初始化章程/技术方案 | `feishu_overwrite_page` | target = `00_项目章程` / `02_技术方案` token |
| 查看项目结构 | `feishu_wiki_page(list_children)` | `parent_wiki_token` = 项目文件夹 token |
| 需要完整文档模板 | 读取 `workspace/SKILLS_PROJECT_MGMT_TEMPLATES.md` | — |

## 三、文档类型 → 目标页面

| 内容类型 | 写入目标 | 写入方式 |
|----------|----------|----------|
| 会议纪要 | `04_会议纪要` 子页面 | 追加 |
| 风险/问题/决策/行动项 | `06_RAID 日志` | 追加对应块 |
| 进展/周报 | `05_状态周报` | 追加（最新在上） |
| 项目章程/范围 | `00_项目章程` | 覆盖写入 |
| 技术方案 | `02_技术方案` | 覆盖写入 |

---

## 四、写入前治理检查

1. **页面归属**：内容属于哪个项目？写到对应项目文件夹下
2. **类型匹配**：按上表选择正确文档
3. **模板格式**：需要完整模板时，读取 `workspace/SKILLS_PROJECT_MGMT_TEMPLATES.md`
4. **结构完整性**：包含模板关键字段，不写碎片信息

---

## 五、特殊关注点

- **多系统接口**：每个集成点必须在 `02_技术方案` 系统交互清单中登记
- **AI 项目**：模型版本、训练数据、评估指标、上线监控必须记录
- **多部门协同**：RACI 矩阵必须填写；跨部门决策记录在 RAID Decisions 块
- **数据治理**：涉及用户/订单数据必须包含数据分级和合规说明
